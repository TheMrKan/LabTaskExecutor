# LabTaskExecutor
### Абстрактная платформа для асинхронной обработки задач
### Предназначена для:
- приёма задач из различных источников;
- хранения и организации задач;
- последовательной и асинхронной обработки задач;
- расширения функциональности без изменения существующего кода.

## Источники задач
Все источники **обязаны** выполнять следующий контракт:
```python
class TaskInitialData(Protocol):
    task_id: str
    payload: Any

class TaskSourceProto(Protocol):
    name: str

    def fetch_new_tasks(self) -> Iterable[TaskInitialData]:
        ...
```
`fetch_new_tasks` должна возвращать ранее не выданные задачи,
т. е. одна и та же задача не может быть возвращена дважды.<br>
`name` используется для предотвращения повторного добавления одного и того же источника.
Чтобы можно было добавить только один источник данного класса, нужно определить `name` как атрибут класса.

### Регистрация источников
Источники задач регистрируются в TaskManager
```python
manager = TaskManager()
file_task_source = FileTaskSource(Path("file.txt"))
manager.add_source(file_task_source)
```
При попытке добавления источника с повторяющимся `name`, будет вызван `KeyError`.
Если объект не выполняет необходимый контракт, то будет вызван `InvalidTaskSourceError`.

### Файловый источник задач (`FileTaskSource`)
Построчно читает задачи из файла в произвольном формате.
Одна строка = одна задача.<br>
Внутри объекта хранит количество ранее прочитанных строк для определения новых задач.

### Источник задач из API эндпоинта (`APITaskSource`)
Запрашивает задачи по HTTP из заданного эндпоинта. Ожидаемый формат ответа:
```json
[
  {
    "task_id": "id",
    "payload": "data"
  },
  ...
]
```
Не отслеживает ранее обработанные задачи - это обязанность эндпоинта.

### Генератор случайных задач (`GeneratorTaskSource`)
Каждый раз генерирует 3 задачи со случайным числом в диапазоне от 0 до 1 в `payload`.
